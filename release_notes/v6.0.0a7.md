# 6.0.0a7

_2026-04-01_

## Highlights

- `Ephemeris trace` is now ordered by canonical body sequence instead of zodiac degree
- The displayed degree remains visible, but no longer determines row order

## Why

Ordering the debug trace by degree made the output jump around from chart to chart and made it harder to scan quickly. During debugging, the useful mental model is usually the standard planetary sequence: Sun, Moon, Mercury, Venus, Mars, and so on.

## Changes

- `AstrologicalSubjectFactory._calculate_planets()` now builds a stable order map from `STANDARD_PLANETS`, `White_Moon`, and `TNO_PLANETS`.
- Trace rows are sorted by canonical point order first, with degree used only as a tie-breaker.
