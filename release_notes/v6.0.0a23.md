# v6.0.0a23

## kr:horoscope attribute for house elements in classic dual charts

Classic dual charts (Transit, Synastry, DualReturnChart) now include `kr:horoscope` on all house-related SVG elements, enabling frontends to distinguish which subject's house was clicked.

### What changed

- **New attribute**: `kr:horoscope="0"` (Subject 1 / inner ring) and `kr:horoscope="1"` (Subject 2 / outer ring) on `Cusp`, `HouseNumber`, and `HouseSector` elements.
- **Subject 2 house sectors**: Transparent interactive sectors are now rendered in the outer ring area (r-36 to r-72) using Subject 2's house cusp boundaries. Clicking in the outer ring area targets Subject 2's houses.
- **Subject 1 house sectors**: The existing sectors (r-72 to r-160) now carry `kr:horoscope="0"`.

### API

`draw_house_sectors()` accepts new optional parameters:

```python
horoscope_id: Union[str, None] = None           # "0" or "1"
seventh_house_abs_override: Union[float, None] = None  # Override wheel orientation
outer_r_offset: Union[int, float, None] = None   # Override outer radius offset
inner_r_offset: Union[int, float, None] = None   # Override inner radius offset
```

`_setup_house_sectors()` accepts an optional `second_houses_list` parameter. When provided in dual charts, both subjects' sectors are rendered with appropriate `kr:horoscope` values.

### Migration

No breaking changes. Existing code continues to work without modification.
