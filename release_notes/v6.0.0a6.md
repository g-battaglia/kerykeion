# 6.0.0a6

_2026-04-01_

## Highlights

- DEBUG logging for ephemeris provenance is now tabular and ordered by degree
- Chart layout debugging now logs only real overlap groups instead of every internal distance check
- All related debug output now comes from module-specific loggers instead of `root`

## Why

The initial backend tracing added in `6.0.0a5` exposed the right information but produced noisy output: one line per body, duplicated rendering details, and low-value spacing diagnostics mixed into the same stream. This release keeps the tracing itself unchanged while making the logs readable enough to use during real debugging sessions.

## Changes

- `AstrologicalSubjectFactory._calculate_planets()` now emits a compact `Ephemeris trace` table with point name, absolute degree, and backend.
- `draw_planets.py` no longer logs every point index or every `distance_to_prev` / `distance_to_next` calculation.
- `draw_planets.py` now logs only grouped overlap clusters, for example `group 1: Pluto(152.20) Asc(154.59) Moon(158.04)`.
- Related debug paths now use module loggers, making it clear whether a message comes from subject calculation, aspect filtering, or chart rendering.
