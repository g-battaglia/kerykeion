# Release Notes v5.12.0

## Highlights

-   **Foundation Enrichment**: Comprehensive expansion of core astrological data — house cusp speeds, 20 new fixed stars (completing all 15 Behenian stars plus 7 bright stars), 27 new sidereal modes, custom ayanamsa support, and new per-point fields (speed, declination, magnitude).
-   **Right-Panel Aspect Layout**: Charts with more than 24 active points automatically render the aspect list to the right of the wheel, preventing excessive vertical growth.
-   **Constructor-Level Chart Style**: `style` and `show_zodiac_background_ring` promoted from render-method-only arguments to `ChartDrawer.__init__()` keyword arguments, enabling per-instance defaults.

## New Features

### House Cusp Speeds

Replaced `swe.houses_ex()` with `swe.houses_ex2()` to expose cusp velocities. All 12 house cusps and the 4 angular points (ASC, MC, DSC, IC) now carry a `speed` field (degrees/day) representing the real rate of diurnal motion. Useful for primary directions and profection techniques.

```python
from kerykeion import AstrologicalSubjectFactory

subject = AstrologicalSubjectFactory.from_birth_data(
    "John Lennon", 1940, 10, 9, 18, 30,
    lng=-2.9833, lat=53.4, tz_str="Europe/London", online=False,
)

# House cusps now include speed
print(subject.first_house.speed)   # e.g. 359.98 (degrees/day)
print(subject.medium_coeli.speed)  # e.g. 361.02
```

Includes all 15 Behenian stars of the medieval/Hermetic tradition and the 4 Royal Stars of Persian/Hellenistic astrology (Regulus, Aldebaran, Antares, Fomalhaut). Each fixed star reports apparent visual magnitude via `swe.fixstar2_mag()` and equatorial declination.

```python
from kerykeion import AstrologicalSubjectFactory

subject = AstrologicalSubjectFactory.from_birth_data(
    "John Lennon", 1940, 10, 9, 18, 30,
    lng=-2.9833, lat=53.4, tz_str="Europe/London", online=False,
)

print(subject.sirius.abs_pos)        # Ecliptic longitude
print(subject.sirius.magnitude)      # -1.44
print(subject.sirius.declination)    # Equatorial declination
print(subject.sirius.speed)          # Daily motion
```

Fixed stars are inactive by default. To include them in chart rendering and aspect calculations, pass their names in the `active_points` parameter:

```python
from kerykeion.chart_data_factory import ChartDataFactory
from kerykeion.settings.config_constants import DEFAULT_ACTIVE_POINTS

chart_data = ChartDataFactory.create_natal_chart_data(
    subject,
    active_points=list(DEFAULT_ACTIVE_POINTS) + ["Sirius", "Regulus", "Aldebaran", "Antares"],
)
```

### Expanded Sidereal Modes

Grew supported ayanamsa systems from 20 to 47 named modes plus a `USER` mode for custom ayanamsa definitions (48 total). New mode families include:

-   **Indian/Vedic variants**: Aryabhata, Suryasiddhanta, True Citra/Pushya/Revati, Lahiri sub-variants
-   **Babylonian systems**: Britton
-   **Galactic alignment systems**: Galactic Center (0° Sagittarius, Mula, Brand)
-   **Astronomical reference frames**: J2000, J1900, B1950
-   **Historical**: Valens Moon ayanamsa

### Custom Ayanamsa (USER Mode)

The `USER` sidereal mode allows defining a custom ayanamsa with a reference epoch and offset:

```python
from kerykeion import AstrologicalSubjectFactory

custom = AstrologicalSubjectFactory.from_birth_data(
    "Custom Ayanamsa", 2000, 1, 1, 0, 0,
    lng=0.0, lat=51.5, tz_str="Etc/GMT", online=False,
    zodiac_type="Sidereal",
    sidereal_mode="USER",
    custom_ayanamsa_t0=2451545.0,      # J2000.0 reference epoch (Julian Day)
    custom_ayanamsa_ayan_t0=23.5,       # Ayanamsa offset at epoch (degrees)
)
```

### Ayanamsa Value Exposure

Added `ayanamsa_value` field to `AstrologicalBaseModel`. For sidereal charts, this contains the computed angular offset (in degrees) between tropical and sidereal 0° Aries at the chart's date/time. `None` for tropical charts.

```python
subject = AstrologicalSubjectFactory.from_birth_data(
    "Example", 1990, 1, 1, 0, 0,
    lng=0.0, lat=51.5, tz_str="Etc/GMT", online=False,
    zodiac_type="Sidereal", sidereal_mode="LAHIRI",
)
print(subject.ayanamsa_value)  # e.g. 23.72
```

### Right-Panel Aspect Layout

Charts with more than 24 active points now render the aspect list/grid to the right of the wheel instead of below it, preventing excessive vertical growth. This is controlled by the internal `_RIGHT_PANEL_POINTS_THRESHOLD` constant and activates automatically.

### Constructor-Level Chart Style

`style` and `show_zodiac_background_ring` promoted from render-method-only arguments (v5.11) to `ChartDrawer.__init__()` keyword arguments. This allows setting a per-instance default that applies to all subsequent render calls, while still permitting per-render overrides:

```python
from kerykeion.charts.chart_drawer import ChartDrawer

# Set style at instance level — all renders will use modern style
chart = ChartDrawer(chart_data=chart_data, style="modern", show_zodiac_background_ring=False)
chart.save_svg(output_path=output_dir)                    # Uses modern, no ring
chart.save_svg(output_path=output_dir, style="classic")   # Per-render override
```

### New Data Fields

All new fields are Optional with `None` default — no breaking changes:

| Field | Model | Description |
|-------|-------|-------------|
| `speed` | `KerykeionPointModel` | Daily motion in degrees/day (planets, house cusps, angular points) |
| `declination` | `KerykeionPointModel` | Equatorial declination in degrees |
| `magnitude` | `KerykeionPointModel` | Apparent visual magnitude (fixed stars only) |
| `ayanamsa_value` | `AstrologicalBaseModel` | Ayanamsa offset in degrees (sidereal charts only) |

## Upgrade Path

This release is fully backward compatible with v5.11.x. All new fields are Optional with `None` defaults — existing code continues to work unchanged. New features are opt-in:

-   To use house cusp speeds, access the `.speed` field on any house cusp or angular point.
-   To use new fixed stars, pass their names in `active_points` to `ChartDataFactory` methods.
-   To use new sidereal modes, pass the mode name to the `sidereal_mode` parameter.
-   To set chart style at instance level, pass `style` and/or `show_zodiac_background_ring` to `ChartDrawer.__init__()`.
