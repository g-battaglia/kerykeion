# v6.0.0a26

## Drop-in replacement of `scour` with `svg-polish`

This alpha swaps the dormant `scour>=0.38.2` SVG optimizer for [`svg-polish>=1.0.0`](https://pypi.org/project/svg-polish/), a hardened, type-safe modernisation of Scour 0.38.2. Output is byte-identical on the kerykeion test matrix; the change is invisible to consumers.

### Why

`scour` 0.38.2 has been dormant upstream since August 2021. It also raises on a class of real-world SVG inputs that `kerykeion` produces — notably CSS `var(--…)`, `calc(…)`, and keyword tokens (`inherit`, `currentColor`) inside `style` / `transform` / `d` / length attributes — forcing the existing fallback path to engage. `svg-polish` handles these gracefully (leaves the attribute untouched and continues), so the minify pipeline now succeeds on inputs where `scour` previously crashed.

`svg-polish` also brings:

- **Secure-by-default XML parsing** via `defusedxml` (rejects XXE, billion-laughs, external DTDs out of the box).
- **Type-safe API** — `OptimizeOptions` frozen dataclass, `OptimizeResult`, full `SvgPolishError` exception hierarchy, `py.typed` marker.
- **Modular internals** (28 single-purpose modules vs scour's 4 700-line monolith) and 700 tests at 100 % line coverage, `mypy --strict` clean.

### What changes for kerykeion users

Nothing observable. The `ChartDrawer.makeSVG(minify=True)` path produces byte-identical output on the chart matrix. The narrow `try/except` around the optimize call is preserved as a defensive fallback for malformed XML — `svg-polish` itself no longer raises on the CSS edge-cases that scour did, but the safety net is kept for truly broken input.

### Internal

- `kerykeion/charts/chart_drawer.py`: `from scour.scour import scourString` → `from svg_polish import optimize as _svg_polish_optimize`. The single call site (`template = scourString(template)` → `template = _svg_polish_optimize(template)`) and the surrounding `try/except` log message are updated accordingly.
- `pyproject.toml`: `scour>=0.38.2` → `svg-polish>=1.0.0`.
- `uv.lock`: `scour` removed; `svg-polish==1.0.0` and its single transitive `defusedxml==0.7.1` added.

### Migration

None. No public API change, no SVG output change, no model schema change.

### Validation

- `pytest tests/`: **8885 / 8885 passed**, 165 skipped (all online-only), 0 failed.
- All chart-drawer tests (natal, synastry, transit, composite, dual-return, ACG, Gauquelin, sidereal modes, topocentric / heliocentric / barycentric, themes, languages, toggles) produce byte-identical SVG output.
