# v6.0.0a35

## Modern HouseSector no longer hides the zodiac click region

A small but high-impact bugfix for any frontend layering interactivity
on top of the modern chart style.

## What was wrong

`_draw_house_sectors_modern` used `R_CUSP_OUTER` (50) as the default
outer radius. The zodiac background ring lives in the annulus from
`R_ZODIAC_BG_INNER` (46) to `R_CUSP_OUTER` (50), so HouseSector and
ZodiacSign painted on top of the same 4-unit ring.

Since both elements declare `pointer-events="all"` inline, a frontend
walking `elementsFromPoint` and visiting HouseSector before ZodiacSign
(the typical priority order: planets → aspects → house numbers →
**house sectors** → gauquelin → zodiac → empty) would always resolve
a click on a zodiac sign as a click on the underlying house.
Symptom: clicking a sign focuses an unrelated house instead.

The visible focused-house tint also bled into the zodiac annulus,
making the highlight look misaligned at its outer edge.

## What changed

When `show_zodiac_background_ring` is true, both the main horoscope
path and the synastry / dual-chart path now pass
`outer_r=R_ZODIAC_BG_INNER` (46) to `_draw_house_sectors_modern`. The
HouseSector annulus stops exactly where the zodiac annulus starts, so
the two regions tile cleanly with zero overlap.

When the zodiac background ring is disabled the original outer radius
(`R_CUSP_OUTER`) is preserved, so charts without a zodiac ring keep
the full-radius house sector.

## Why this is the right boundary

`R_ZODIAC_BG_INNER` is the same constant the zodiac wedge uses for its
inner arc, so cap-on-cap there is one continuous radial boundary
between the two click regions. No fractional gap, no wasted dead-zone.

## Backward compatibility

- Visible geometry unchanged: HouseSector is invisible by default.
- API unchanged: `_draw_house_sectors_modern` keeps the same signature
  (the public default remains `R_CUSP_OUTER`); only the internal call
  sites pass the clipped value.
- Frontend code that ignored the overlap (e.g. relied solely on
  `data-kr-node="ZodiacSign"` for sign clicks) keeps working.

## Coverage

| Path | Updated | Notes |
|---|---|---|
| `_draw_modern_horoscope_chart_main` | yes | natal / composite / return |
| `_draw_modern_dual_horoscope_chart` | yes | synastry / transit / dual return |
| `_draw_house_sectors_modern` signature | unchanged | callers pass the new outer_r |
| Classic chart paths | not affected | classic HouseSector already stops at the planet ring |
