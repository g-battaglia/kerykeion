# v6.0.0a40

## Clean ephemeris packaging and Swiss Ephemeris setup utility

This alpha finalizes the separation between the Kerykeion package and
the optional Swiss Ephemeris backend. The default backend (`libephemeris`)
continues to work out of the box with no external data files.

## What's new

### `python -m kerykeion.swisseph_setup`

Users who want the Swiss Ephemeris backend can now set it up with a single
command:

```bash
pip install kerykeion[swiss]
python -m kerykeion.swisseph_setup
```

The utility:
- Displays the Swiss Ephemeris license terms (AGPL-3.0, Astrodienst AG)
  and asks for explicit confirmation
- Downloads the main ephemeris files (planets, Moon, asteroids, fixed stars)
  from the official Swiss Ephemeris repository
- Provides instructions for optional TNO/dwarf planet files
- Prints the environment variables to set

```bash
# Non-interactive mode (CI/scripts)
python -m kerykeion.swisseph_setup --yes

# Custom directory
python -m kerykeion.swisseph_setup --target /path/to/data
```

### Backend-aware ephemeris path

`EPHE_DATA_PATH` is now resolved per-backend:

- **libephemeris**: empty string (manages its own data via `~/.libephemeris/leb/`)
- **swisseph**: empty string by default (Moshier fallback), or the path set
  via `KERYKEION_EPHE_PATH`

When swisseph is active and the user-provided path lacks `.se1` files,
a validation warning is logged.

### Documentation

- New **Swiss Ephemeris Configuration** guide
  (`site/docs/swisseph_configuration.md`) covering setup, data files,
  and license terms
- Updated README with Swiss Ephemeris backend section

### Packaging

- Swiss Ephemeris data files are no longer included in the wheel
- `MANIFEST.in` no longer includes `.se1` / `.txt` data files
- Defensive `exclude` rule in hatch wheel config
- PyPI license classifier corrected to AGPL-3.0

## Backward compatibility

- `EPHE_DATA_PATH` defaults to `""` instead of a package-internal directory.
  All factory modules handle this correctly.
- The `[swiss]` optional extra and `KERYKEION_BACKEND=swisseph` env var
  continue to work as before; the only difference is that data files
  must now be provided by the user via `KERYKEION_EPHE_PATH`.
