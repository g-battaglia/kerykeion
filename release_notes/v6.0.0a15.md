# 6.0.0a15

_2026-04-07_

## Highlights

- Enriched SVG chart metadata with `kr:` namespace attributes on all interactive elements
- Enables frontend focus mode (click planet → highlight its aspects) and DataCards (planet info dialogs)
- Zero breaking changes — additive metadata only

## SVG Metadata Enhancements

- **Degree indicators**: Wrapped in `<g kr:node="Indicator" kr:slug="{planet}">` (both primary outer-ring and inner dual-chart variants). Frontend can now dim/show indicators alongside their planet.
- **Transit planet glyphs**: Added full `kr:node="ChartPoint"` metadata (`kr:slug`, `kr:house`, `kr:sign`, `kr:absoluteposition`, `kr:signposition`). Previously only had `class="transit-planet-name"` with no queryable metadata.
- **House numbers**: Added `kr:house="{n}"` attribute to `<g kr:node="HouseNumber">` elements. Frontend can now query house numbers by attribute instead of parsing `<tspan>` text content.
- **Connecting lines**: Wrapped external-view connecting lines in `<g kr:node="ConnectingLine" kr:slug="{planet}">`.

## Fixes

- Fixed typo `kr:sing` → `kr:sign` on Cusp elements (both first and second subject house cusps).

## Test Baselines

- Regenerated all SVG test baselines to reflect the new metadata attributes.

## Complete `kr:` Attribute Reference

After this release, the SVG contains the following queryable element types:

| `kr:node` value | Key attributes | Description |
|----------------|---------------|-------------|
| `ChartPoint` | `kr:slug`, `kr:house`, `kr:sign`, `kr:absoluteposition`, `kr:signposition`, `kr:retrograde` | Planet/point glyphs (natal and transit) |
| `Aspect` | `kr:aspectname`, `kr:from`, `kr:to`, `kr:orb`, `kr:aspectdegrees`, `kr:aspectmovement` | Aspect lines between planets |
| `HouseNumber` | `kr:house` | House number labels (1-12) |
| `Cusp` | `kr:slug`, `kr:absoluteposition`, `kr:signposition`, `kr:sign` | House cusp lines |
| `Indicator` | `kr:slug` | Degree position indicators (line + text) |
| `ConnectingLine` | `kr:slug` | External-view connecting lines |
