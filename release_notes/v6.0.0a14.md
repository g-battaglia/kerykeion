# 6.0.0a14

_2026-04-03_

## Highlights

- Performance optimizations across chart rendering and subject computation
- Benchmark tooling (`poe benchmark`) for tracking performance
- Complete baseline regeneration for all configuration-specific test data

## Performance

- **SVG rendering**: Cache templates and CSS themes with `@lru_cache`; convert string concatenation to list-join pattern in drawing functions.
- **Subject computation**: Consolidate `model_copy()` calls in optional calculations (dignities, nakshatras, gauquelin, local space, OOB) from up to 345 intermediate Pydantic models down to max 69.
- **Ephemeris session**: Use `reset_session()` instead of `close()` to preserve LEB reader, Skyfield timescale, SPK kernels, and LRU caches across consecutive calculations.
- **Language settings**: Cache `load_language_settings()` for the common no-overrides case.

## Tooling

- Added `poe benchmark` script measuring subject creation, aspects calculation, and SVG rendering.
- Added `poe regenerate:configurations` covering sidereal modes, perspectives, house systems, returns, composite, ephemeris, and arabic parts.
- Included `regenerate:configurations` in `poe regenerate:all` so no configuration type is missed.

## Fixes

- Regenerated all configuration-specific baselines for libephemeris 1.0.0a15.
- Relaxed Pluto-Chiron aspect movement test to accept both Static and Applying (slow planet boundary sensitivity).
