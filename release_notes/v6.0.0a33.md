# v6.0.0a33

## Glyph-center metadata extended to modern style

This alpha completes the work started in 6.0.0a32 by emitting `kr:cx` and
`kr:cy` on every ChartPoint produced by the **modern** chart style too.
Both styles now expose the rendered glyph center as a pair of attributes
in chart SVG root coords, so a frontend that layers interactivity on top
of kerykeion's output needs only one code path for hit-detection across
classic and modern.

## What changed

Modern-style ChartPoints wrap their content in
`<g transform="rotate(-display_angle, CENTER, CENTER)">`, and the glyph
sits at `(CENTER, glyph_y)` in the pre-rotation local coord system. The
rotation has to be applied before the coordinates are emitted. Closing
the form gives:

```
cx = CENTER + (glyph_y - CENTER) * sin(display_angle)
cy = CENTER + (glyph_y - CENTER) * cos(display_angle)
```

Values are computed in `_draw_single_planet_in_ring` before the opening
`<g>` is assembled, then written alongside the existing `kr:*`
attributes.

## Why

6.0.0a32 added `kr:cx` / `kr:cy` only on the classic path. A frontend
targeting both styles would therefore still have to parse the modern
transform chain — defeating much of the point of emitting the center in
the first place. With this release both styles are covered and the
consumer contract is uniform.

## Usage (unchanged)

```javascript
// Same two-line recipe as 6.0.0a32 — works on classic AND modern now.
const cx = parseFloat(chartPoint.getAttribute('kr:cx'))
const cy = parseFloat(chartPoint.getAttribute('kr:cy'))
const screenPt = new DOMPoint(cx, cy).matrixTransform(
  chartPoint.ownerSVGElement.getScreenCTM()
)
// screenPt is the viewport-pixel glyph center.
```

For consumers that need the center in the ChartPoint's local coord system
(e.g. to inject overlays inside the `<g>`), convert via the inverse of
`getCTM()`:

```javascript
const local = new DOMPoint(cx, cy).matrixTransform(
  chartPoint.getCTM().inverse()
)
```

## Coverage summary

| Style   | Path                                    | Attributes from |
|---------|-----------------------------------------|-----------------|
| classic | `draw_planets._generate_point_svg`      | 6.0.0a32        |
| classic | transit inline in `draw_planets`        | 6.0.0a32        |
| modern  | `draw_modern._draw_single_planet_in_ring` | **6.0.0a33 (this release)** |

## Backward compatibility

Additive only. No existing attribute is removed, renamed, or
semantically changed. SVG geometry is byte-identical. Consumers that
ignore `kr:cx` / `kr:cy` are unaffected.
