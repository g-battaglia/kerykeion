# 6.0.0a13

_2026-04-03_

## Highlights

- Test baselines regenerated for libephemeris 1.0.0a15
- Cross-backend test tolerances fixed (swisseph now gets relaxed tolerances)

## Why

The test baselines were generated with an older libephemeris version and an older point calculation order. Two issues caused test failures:

1. **Point ordering changed** in v6.0.0a3: Descendant, Imum Coeli, and True South Lunar Node moved from inline/house calculation to `OPPOSITE_PAIRS`, changing the order of `active_points` and the house comparison table in synastry/transit SVGs.
2. **libephemeris 1.0.0a15** introduced minor precision differences (~0.001°) in planetary positions and house cusps.
3. **Tolerance logic was inverted**: the code assumed baselines were generated with swisseph, giving swisseph tight tolerances when comparing against libephemeris baselines.

## Changes

- Regenerated all SVG, report, and expected-data test baselines with libephemeris 1.0.0a15.
- Inverted all `BACKEND_NAME != "swisseph"` tolerance conditions to reflect that baselines are now generated with libephemeris.
- Increased swisseph position tolerance from 0.15° to 0.2° (ancient 500 BC ΔT model divergence).
- Skipped heliocentric synastry SVG test for swisseph (house comparison numbers differ across backends).
