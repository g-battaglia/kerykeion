# v6.0.0a34

## Sign-full highlight overlay on modern ZodiacSign

This alpha adds an invisible full pie-slice overlay to every ZodiacSign
in the modern chart style, so frontends can render a focused sign with
the same visual impact as the classic wheel — without needing their own
arc geometry.

## What changed

In `draw_modern._draw_zodiac_background_ring`, each `<g kr:node="ZodiacSign">`
now includes a second `<path>`:

```xml
<path kr:highlight="sign-full"
      d="M <center> L <outer_start> A <outer arc> Z"
      fill="<sign color>"
      fill-opacity="0"
      pointer-events="none" />
```

- Geometry: a full pie slice from the chart center (50, 50) to the outer
  zodiac boundary (R_ZODIAC_BG_OUTER = 50).
- Opacity: invisible by default; the consumer flips it on via CSS (for
  example `.chart-focused > [kr:highlight="sign-full"] { fill-opacity: 0.15 }`)
  when its ZodiacSign group is focused.
- Interactivity: `pointer-events: none` — the overlay never steals
  clicks from the planet ring, aspect core, or house sectors sitting
  above it in z-order.
- Fill: reuses the existing per-sign CSS variable
  `--kerykeion-modern-zodiac-bg-<N>` so the highlight tint matches the
  visible outer-ring color.

## Why

The modern layout draws each ZodiacSign as a thin outer annular wedge
(R_ZODIAC_BG_INNER = 46 to R_ZODIAC_BG_OUTER = 50 — 4 units on a
100-unit viewBox). The classic layout draws a full pie-slice wedge.
A consumer using the same "highlight focused wedge" pattern therefore
gets a broad orange-juice slice on classic and a hairline arc on modern.

The new overlay gives the modern consumer the option of a full-slice
highlight without changing the default visual style and without having
to replicate kerykeion's arc math in the frontend.

## Coverage summary

| Path | Emits overlay | Since |
|---|---|---|
| `draw_modern._draw_zodiac_background_ring` | yes | **6.0.0a34 (this release)** |
| `draw_planets` / classic zodiac slice | n/a (already full pie slice) | — |

## Usage

```css
/* Light the focused sign's full wedge */
[data-kr-node='ZodiacSign'].chart-focused [data-kr-highlight='sign-full'] {
    fill-opacity: 0.18;
}
.dark [data-kr-node='ZodiacSign'].chart-focused [data-kr-highlight='sign-full'] {
    fill-opacity: 0.22;
}

/* Keep it hidden when the sign is dimmed */
[data-kr-node='ZodiacSign'].chart-dimmed [data-kr-highlight='sign-full'] {
    fill-opacity: 0;
}
```

## Backward compatibility

Additive only. The visible outer-wedge path is unchanged, no existing
attributes are removed or renamed, and SVG snapshots compare identical
except for the extra overlay path per sign. Consumers that ignore
`kr:highlight` are unaffected.
